# 代码阅读指南

## 📋 文档说明

本文档旨在帮助开发者快速理解项目代码结构，提供推荐的文件阅读顺序和学习路线。

---

## 🏗️ 项目架构概览

```
AI小说创作桌面应用（AIWT）
├── Electron 主进程（后台）    ← 入口：electron/main.ts
├── Vue3 渲染进程（前端）     ← 入口：src/main.ts
└── 共享类型定义                ← 桥梁：TypeScript interfaces
```

**核心设计理念**：
- **分层架构**：UI层 → 状态层 → 业务层 → 数据层
- **类型优先**：所有接口都有完整的TypeScript类型定义
- **模块化**：每个功能都有独立的文件和清晰的职责

## 📖 详细文件说明

### 1. LLM适配层

#### `src/llm/types.ts`

**作用**：定义LLM适配层的所有类型接口

**导出内容**：
- `LLMProviderType` - 模型提供商枚举
- `ModelConfig` - 模型配置接口
- `LLMMessage` - 消息格式接口
- `LLMAdapter` - 适配器接口（核心）

**依赖**：无

**被依赖**：所有LLM相关文件

**阅读重点**：
- `LLMAdapter`接口定义了所有适配器必须实现的方法
- `ModelConfig`包含模型配置的所有字段
- `LLMMessage`统一了消息格式

#### `src/llm/LLMClient.ts`

**作用**：提供统一的LLM调用入口

**核心方法**：
- `chat(messages, config)` - 非流式调用
- `stream(messages, config)` - 流式调用
- `healthCheck(config)` - 健康检查

**依赖**：`types.ts`, `LLMProviderFactory.ts`

**阅读重点**：
- 如何根据配置选择适配器
- 如何处理错误和重试
- 如何统一返回格式

#### `src/llm/LLMProviderFactory.ts`

**作用**：工厂类，根据配置创建对应的适配器

**核心方法**：
- `createAdapter(config)` - 创建适配器

**依赖**：`types.ts`, `adapters/*`

**阅读重点**：
- 工厂模式实现
- 如何扩展新的适配器

#### `src/llm/adapters/OpenAIAdapter.ts`

**作用**：OpenAI模型适配器（也用于DeepSeek、通义等兼容API）

**核心方法**：
- `chat(messages, config)` - 实现聊天接口
- `stream(messages, config)` - 实现流式接口
- `healthCheck(config)` - 实现健康检查

**依赖**：`types.ts`, `openai` SDK

**阅读重点**：
- 如何使用openai-node SDK
- 如何处理流式响应（`AsyncGenerator`）
- 如何转换消息格式

### 2. Agent系统

#### `src/agents/types.ts`

**作用**：定义Agent系统的所有类型接口

**导出内容**：
- `AgentType` - Agent类型枚举（17种）
- `AgentInput` - Agent输入（Tagged Union）
- `AgentOutput` - Agent输出
- `AgentContext` - Agent执行上下文

**依赖**：无

**阅读重点**：
- `AgentInput`使用Tagged Union（`type`字段区分）
- 每种Agent输入都有明确的类型定义
- 避免使用`any`类型

#### `src/agents/base.ts`

**作用**：Agent抽象基类，定义所有Agent的通用行为

**核心方法**：
- `execute(input)` - 执行Agent（抽象）
- `stream(input)` - 流式执行（抽象）
- `buildContext(project, options)` - 构建上下文（可重写）
- `mountSkill(skill)` / `unmountSkill(skillId)` - Skill管理

**依赖**：`types.ts`, `llm/LLMClient`

**阅读重点**：
- 模板方法模式：`buildContext`提供默认实现
- 异步生成器：`stream`返回`AsyncGenerator<string>`
- Skill扩展：通过`mountSkill`动态扩展能力

#### `src/agents/IdeaAgent.ts`

**作用**：创意激发Agent（示例实现）

**核心方法**：
- `execute(context)` - 执行创意激发
- `stream(context)` - 流式输出

**依赖**：`base.ts`

**阅读重点**：
- 如何继承`BaseAgent`
- 如何构建提示词
- 如何调用LLM

4. **直接上手修改**：通过实践理解代码

### 常见问题

**Q1：为什么使用Tagged Union而不是`any`？**  
A1：Tagged Union提供编译时类型检查，避免使用`any`导致的运行时错误。

**Q2：为什么使用`AsyncGenerator`实现流式输出？**  
A2：`AsyncGenerator`可以逐个产出token，实现真正的流式输出，提升用户体验。

**Q3：为什么要分片存储章节？**  
A3：避免单大JSON文件性能问题，便于版本快照管理。
