### 5. 工具函数和扩展（2个文件）

#### 5.1 上下文构建器

**文件**：`src/utils/contextBuilder.ts`

**核心功能**：
```typescript
export async function buildContext(options: ContextOptions): Promise<string> {
  const { project, chapterId, maxTokens = 8000 } = options
  
  // 1. 计算各模块token预算
  const budgets = calculateBudgets(project, maxTokens)
  
  // 2. 组装上下文
  const parts: string[] = []
  
  // 世界观设定（预算：20%）
  if (project.worldSettings) {
    parts.push(`# 世界观\n${truncate(project.worldSettings, budgets.world)}`)
  }
  
  // 角色设定（预算：30%）
  const relevantChars = findRelevantCharacters(project, chapterId)
  parts.push(`# 相关角色\n${formatCharacters(relevantChars, budgets.character)}`)
  
  // 前章摘要（预算：20%）
  const prevSummary = await loadPreviousSummary(project, chapterId)
  if (prevSummary) {
    parts.push(`# 前章摘要\n${prevSummary}`)
  }
  
  // 本章大纲（预算：10%）
  const outline = findOutlineForChapter(project, chapterId)
  if (outline) {
    parts.push(`# 本章大纲\n${outline}`)
  }
  
  // 3. 拼接并返回
  return parts.join('\n\n')
}
```

**设计亮点**：
- 动态预算分配：根据各模块重要性分配token
- 相关性过滤：只注入与当前章节相关的角色
- 截断保护：超过预算自动截断，保证总token不超限

### 2. 类型安全设计

**问题**：JavaScript动态类型导致运行时错误  
**解决**：全项目TypeScript + 严格类型定义

```typescript
// ❌ 错误示例 - 使用any
function execute(agent: any, input: any): Promise<any> {
  // 无法知道agent有哪些方法
  // 无法知道input有哪些字段
  // 编译时无法发现错误
}

// ✅ 正确示例 - 严格类型
function execute(agent: BaseAgent, input: AgentInput): Promise<AgentOutput> {
  // 编译时检查：
  // - agent必须有execute方法
  // - input必须符合AgentInput类型
  // - 返回值必须是AgentOutput
}
```

### 3. 流式输出设计

**问题**：AI生成内容需要等待完整响应，用户体验差  
**解决**：使用AsyncGenerator实现逐字流式输出

```typescript
// LLM适配器返回AsyncGenerator
async *stream(messages: LLMMessage[]): AsyncGenerator<string> {
  const response = await fetch('/api/chat', { ... })
  const reader = response.body.getReader()
  
  while (true) {
    const { done, value } = await reader.read()
    if (done) break
    yield decodeToken(value)  // 逐个token产出
  }
}

// Agent层透传AsyncGenerator
async *execute(input: AgentInput): AsyncGenerator<string> {
  const messages = this.buildMessages(input)
  yield* this.llmClient.stream(messages)
}

// UI层逐字显示
for await (const token of agent.execute(input)) {
  editor.commands.insertToken(token)
}
```

**设计优势**：
- 用户看到实时生成过程，体验大幅提升
- 可以中途取消生成（通过break）
- 内存占用小（不需要等待完整响应）

### 4. 任务队列设计

**问题**：多个Agent同时调用LLM，导致：
- Token预算超限
- 状态混乱
- UI卡顿

**解决**：AgentQueueManager统一管理

```typescript
class AgentQueueManager {
  private queue: AgentTask[] = []
  private runningTasks: Set<string> = new Set()
  private maxConcurrency = 3
  
  enqueue(task: AgentTask): void {
    // 1. 根据优先级插入队列
    if (task.priority === 'high') {
      this.queue.unshift(task)
    } else {
      this.queue.push(task)
    }
    
    // 2. 触发处理
    this.processQueue()
  }
  
  private async processQueue(): Promise<void> {
    // 并发控制
    while (this.runningTasks.size < this.maxConcurrency && this.queue.length > 0) {
      const task = this.queue.shift()!
      this.runningTasks.add(task.id)
      
      // 异步执行，不阻塞
      this.executeTask(task).then(() => {
        this.runningTasks.delete(task.id)
        this.processQueue()  // 执行完毕，处理下一个
      })
    }
  }
}
```

**设计优势**：
- 并发控制：最多同时运行3个任务
- 优先级调度：高优先级任务优先执行
- 状态同步：通过Pinia store实时更新UI

### 5. 上下文管理设计

**问题**：LLM上下文窗口有限（通常4k-128k token），需要合理分配  
**解决**：动态预算分配 + 相关性过滤

```typescript
function buildContext(options: ContextOptions): string {
  const { project, chapterId, maxTokens = 8000 } = options
  
  // 1. 计算预算
  const budgets = {
    world: Math.floor(maxTokens * 0.2),      // 世界观 20%
    characters: Math.floor(maxTokens * 0.3),  // 角色 30%
    summary: Math.floor(maxTokens * 0.2),     // 前章摘要 20%
    outline: Math.floor(maxTokens * 0.1),     // 大纲 10%
    reserved: Math.floor(maxTokens * 0.2)     // 保留 20% for 输出
  }
  
  // 2. 过滤相关角色（只注入当前章节需要的）
  const relevantChars = project.characters.filter(char => 
    char.appearsInChapters.includes(chapterId)
  )
  
  // 3. 组装（超预算自动截断）
  const parts = []
  parts.push(truncate(project.worldSettings, budgets.world))
  parts.push(formatCharacters(relevantChars, budgets.characters))
  // ...
  
  return parts.join('\n\n')
}
```

**设计优势**：
- 动态调整：根据总预算自动分配各模块
- 相关性过滤：只注入必要信息，避免浪费token
- 截断保护：超过预算自动截断，保证不超限
